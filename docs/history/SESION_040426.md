## 1. Tab Recursos — Mejoras completadas

### Problema inicial
Los recursos no se mostraban en la UI (aparecía la lista vacía). La causa era que la BD tenía un array vacío `[]` guardado de una sesión anterior con código roto.

### Fix de estructura de datos
- Componente `PilarRecursos` ahora maneja **3 formatos posibles** de datos:
  - Array directo `[...]`
  - Objeto anidado `{recursos: [...]}`
  - Array vacío o null → fallback a `RECURSOS_DEFAULT`
- Si el array está vacío, siempre muestra los 8 recursos por defecto

### Funcionalidades implementadas
| Feature | Estado |
|---------|--------|
| Todos los toggles en OFF por defecto | ✅ |
| Input de costo deshabilitado cuando toggle OFF | ✅ |
| Auto-reset costo a $0 al desactivar toggle | ✅ |
| Validación solo números >= 0 | ✅ |
| Sin botón eliminar en recursos `obligatorio:true` | ✅ |
| **Suscripción Optima** como recurso obligatorio | ✅ |
| Valor de Optima readonly (calculado desde plan) | ✅ |
| Conversión de moneda según país del perfil | ✅ |
| Total mensual con símbolo y código de moneda dinámico | ✅ |

### Constantes de conversión de moneda (en ProyectoLaboral.jsx)
```js
PRECIO_OPTIMA_MXN = { free: 0, semanal: 99, mensual: 249, trimestral: 659 }
TASAS_DESDE_MXN   = { MXN:1, USD:0.059, EUR:0.054, COP:232, ARS:17.2, ... }
```

### Lógica de % de Recursos
- **100%** solo si: Suscripción Optima activa + mínimo 3 recursos más activos
- Si no se cumple: 0%

## 2. Refactor — Única fuente de verdad para progreso

### Archivo creado
`frontend/src/utils/progresoLaboral.js`

Contiene funciones puras sin dependencias React:
- `RECURSOS_DEFAULT` — array de 8 recursos con toggles OFF
- `calcPerfilPts(perfil, jpData)` — puntos del pilar Mi Perfil
- `calcularProgreso(data, perfil)` — progreso total 0-100

**Por qué:** AuthContext y ProyectoLaboral necesitaban la misma lógica de cálculo. Antes había código duplicado que podía divergir.

## 3. Flujo de Onboarding y Feature Locking

### Guards de rutas implementados en App.jsx

| Componente | Comportamiento |
|------------|----------------|
| `PrivateRoute` | No autenticado → `/auth` |
| `OnboardingGuard` | Autenticado + onboarding pendiente → `/bienvenida` |
| `BienvenidaRoute` (**NUEVO**) | No autenticado → `/auth` \| Ya completó onboarding → `/dashboard` |

### Rutas corregidas
```
/bienvenida      → BienvenidaRoute (antes sin guard)
/proyecto-laboral → PrivateRoute (antes sin guard)

```

## 5. Sidebar — Feature Locking

### Dos niveles de bloqueo
| Estado | Componente | Visual |
|--------|------------|--------|
| `locked` (onboarding) | `LockedNavItem` | Gris, sin tooltip |
